Istio版本 >=1.5

安装

## kubernetes上安装istioctl

##1. istio安装 profile为demo的istio 集群
istioctl manifest apply --set profile=demo
##2. 修改ingeress 如果没有loadbalance 则修改为 nodeport
kubectl patch svc istio-ingressgateway -nistio-system  -p '{"spec":{"type":"NodePort"}}'
##3. kubectl get all -nistio-system
##4. 卸载istio
istioctl manifest generate --set profile=demo | kubectl delete -f -

Istioctl 使用

profile

istioctl profile list
istioctl profile dump default >default.yaml 
## 一般对端口修改的时候可以使用这样的方式dump下来,修改完之后执行
## istioctl manifest apply -f defaul.yaml

Inject

解释：Inject Envoy sidecar into Kubernetes pod resources（针对service ，secrete,configmap 资源不起作用）。

  • 主动注入envoy sidecar
istioctl kube-inject -f deploynginx.yaml |kubectl apply -f - -n default
  • 自动注入
## 给命名空间打入标签 istio-injection=enable, 则在此命令空间下的资源，都会被自动注入sidecar。
istioctl label ns  jixu istio-injection=enabled

  • istio-proxy 运行原理

​

如图： nginx Pod中会同时启动三个容器分别是：

  • initContainer

    • 执行 istio-iptables脚本创建istio的四个链路，创建完之后退出

  • Istio-Proxy(istio-pilot,envoy)

    [root@k8s-slave01 ~]# kubectl exec -it  nginx-64759d697c-mgxqb -c istio-proxy -- ps -ef
UID        PID  PPID  C STIME TTY          TIME CMD
istio-p+     1     0  0 08:12 ?        00:00:00 /usr/local/bin/pilot-agent proxy
istio-p+    19     1  0 08:12 ?        00:00:01 /usr/local/bin/envoy -c etc/isti
istio-p+    51     0  0 08:15 pts/0    00:00:00 ps -ef
    • 运行istio–agent,根据pilot-discovery 同步数据（pilot-discovery是运行在k8s istio-system下的istiod服务端，它watch kube-apiserver的变化，当用户对k8s资源进行操作时，istiod会同步 api-server的资源，istio-proxy中的istio-agent容器sync istio-disvoery的配置规则，istio-agent将配置文件策略和行为告诉envoy，从而实现对流量控制和行为管理）
    • envoy 是一个轻量级的代理服务。

  • nginx Container

    • web业务容器

注意小技巧：initContainer和istio-proxy用的时同一个镜像，而initContainer执行完istio-iptables脚本之后，就退出，Iistio-agent一直运行是因为,

镜像中的entypoint是

 "Entrypoint": [
                "/usr/local/bin/pilot-agent"

initContainers 中使用的command

    enableServiceLinks: true
    initContainers:
    - Commnads:
      - istio-iptables

k8s 中的command覆盖dockerfile中entryPoint， 不过istio 1.65不再使用这种方式，initContainer直接是args，可能对命令做了封装。

Istio Traffic Management

流量管理简单点就是：使用策略控制流量的流向和大小。

同时，业务代码只需要关注业务，超时和重连等机制直接交给istio流量管理即可。

Istio Architecture：（istio 1.5之后）

  • 数据面流量：业务之间调用的流量
  • 控制面流量：istio各组件之间配置和控制网格行为的流量

Istio virtual service

  • 概念： virtual service 是一个抽象概念，简单点可以认为是一个virtual service 是一个类似nginx.conf中location 这样的片段文件

  • 配置：

    • hosts filed： 可配置为 短域名（servicename),长域名(servicename.namesapce.default.local.cluster) 和ingress controller
